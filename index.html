What is a Database?

Database: A collection of related data stored in a manner that enables information to be retrieved as needed.

DBMS (Database Management System): Software used to create, maintain, and access databases.

Database Components

  • Tables: Collection of related records
  • Fields/Columns: Single category of data
  • Records/Rows: Collection of related fields

Primary Key

A field that uniquely identifies records in a table and relates tables to each other.

Database Personnel

  • Database Designers
  • Database Developers
  • Database Programmers
  • Database Administrators
  • Users

Data Concepts & Characteristics

Data Hierarchy

  • Fields → Records → Tables → Database

Entities & Relationships

  • Entity: Person, object, or event of importance
  • Attributes: Characteristics of an entity
  • Relationship: Association between entities

Relationship Types

  • One-to-One (1:1): Not common
  • One-to-Many (O:M): Most common
  • Many-to-Many (M:M): Complex relationships

Data Definition

Process of describing properties of data:

  • Name (unique within table)
  • Data type (Text, Number, Currency, etc.)
  • Description
  • Properties (field size, format, range, etc.)

Data Integrity, Security & Privacy

Data Integrity

  • Accuracy of data
  • Data validation ensures entered data is valid
  • Database locking prevents concurrent changes

Data Security

  • Protects against destruction and misuse
  • Prevents unauthorized access
  • Includes backup and disaster-recovery procedures

Data Privacy

  • Growing concern with vast personal data storage
  • Breach notification laws in many states
  • Costly breaches (approx. $200 per record)

Database Classifications

Single-User vs. Multiuser

Single-User: One computer, one user

Multiuser: Multiple users (most business databases)

Client-Server & N-Tier

Client-Server: Front-end clients, back-end server

N-Tier: Additional middleware layers

Centralized vs. Distributed

Centralized: Single computer location

Distributed: Data divided among networked computers

Disk-Based vs. In-Memory

Disk-Based: Data stored on hard drives

In-Memory: Data stored in main memory (faster)

Database Models

Relational Database Model (RDBMS)

  • Most widely used today
  • Data organized in tables related by common fields
  • Design process: Purpose → Tables/Fields → Normalization → Structure

Object-Oriented Database Model

  • Stores multiple data types as objects with related code
  • Can contain video, text with music, etc.
  • Uses Object Query Language (OQL)

Hybrid Database Models

  • Combination of two or more database types
  • Example: Hybrid XML/Relational Database

Multidimensional Databases (MDDB)

  • Used with data warehousing
  • Used with Online Analytical Processing (OLAP)
  • Types: MOLAP, ROLAP, HOLAP

Cloud Databases

Hosted on cloud provider's servers, accessible via the Web.

Uses

  • Information retrieval for Web pages
  • E-commerce (product info, pricing, shopping carts)
  • Enable dynamic Web pages

How They Work

  1. User makes request via Web form
  2. Web server converts request to database query
  3. Database server performs query
  4. Results sent back to user via Web page

Middleware

Software connecting Web server and DBMS:

  • JavaScript, VBScript
  • CGI Scripts, Active Server Pages (ASPs)
  • PHP Scripts

Database Evolution

Databases have evolved from simple flat files to sophisticated relational, object-oriented, and multidimensional systems.

Model Year Began Data Organization Skill Level Required
Flat Files 1940s Flat files Programmer
Hierarchical 1960s Trees Programmer
Network 1980s Trees Programmer
Relational 1970s Tables and relations User
Object-Oriented 1980s Objects User
Multi-dimensional 1990s Data cubes User

DBMS Advantages & Disadvantages

Advantages

  • Low redundancy
  • Faster response time
  • Lower storage requirements
  • Easier to secure
  • Increased data accuracy

Disadvantages

  • Increased vulnerability
  • Backup is essential

File Management Systems have higher redundancy as tables are not related and cannot retrieve data from multiple tables at once.

Quick Quiz

1. A column in a database in which customer names are stored would be referred to as a ______.

a) field   b) record   c) table

Answer: a) field

2. True or False: Data validation procedures are used to ensure that data entered into a database matches the specified type, format, and allowable value.

Answer: True

3. The ______ contains metadata about the database tables in a database.

Answer: data dictionary

4. Which type of database system is beginning to be used in high-end systems where performance is crucial?

a) In-memory databases   b) Disk-based databases   c) Single-user databases

Answer: a) In-memory databases

5. Which of the following is the most widely used type of database today?

a) Network   b) Relational   c) Object-oriented

Answer: b) Relational
